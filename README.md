## Что это и как работает

Epic-Claimer заходит в ваш аккаунт Epic Games через настоящий браузер (Camoufox/Playwright), сам проходит проверку hCaptcha с помощью ИИ (`hcaptcha-challenger` + Google Gemini) и забирает текущие бесплатные игры в библиотеку. Запускается в Docker, может работать постоянно по расписанию и обслуживать несколько аккаунтов.

Один прогон делает следующее:

1. Логинится в Epic (или переиспользует сохранённую сессию).
2. При необходимости решает hCaptcha и **отказывается** от предложения подключить 2FA.
3. Получает список бесплатных игр недели и сверяет с вашей библиотекой.
4. Забирает те, которых ещё нет (через кнопку **Get → Add to library**).
5. Если забирать нечего — корректно завершает работу.

## Что нужно

| Требование | Зачем |
| --- | --- |
| **Аккаунт Epic Games с выключенной 2FA** | Двухфакторная аутентификация ломает автоматический вход. Отключите её в настройках Epic перед запуском. |
| **API-ключ Google Gemini** | Решает hCaptcha. Бесплатный ключ: [Google AI Studio](https://aistudio.google.com/apikey) → **Create API Key**. См. важный нюанс про лимиты ниже. |
| **Docker + Docker Compose** | Запуск. Образ собирается локально из исходников этого репозитория. |

## ⚠️ Важный нюанс: лимиты Gemini

Это самое важное, что нужно понимать перед запуском.

- **`gemini-2.5-pro` недоступна на бесплатном тарифе** (квота = 0). Поэтому решатель капчи по умолчанию настроен на **`gemini-2.5-flash`** — с бесплатным ключом всё работает «из коробки».
- **Бесплатный тариф жёстко ограничен по числу запросов в день** (в нашем тесте ключ упирался в ~20 запросов/сутки на flash). Одно решение логин-капчи может потребовать **несколько** запросов к Gemini (а при сложной капче — ещё и повторов), так что дневной лимит реально исчерпать за один-два «холодных» входа.
- **Но!** После первого успешного входа **сессия сохраняется** (в `volumes/user_data/<email>`), и при следующих запусках вход капчу **не запрашивает** — ежедневный сбор игр идёт почти без обращений к Gemini. Дорогой по квоте только **первый/холодный** вход.

**Вывод:**
- Для **личного** использования бесплатного ключа обычно достаточно: один раз вошли (возможно, со второй попытки из-за квоты), дальше работает на сохранённой сессии.
- Для **надёжного холодного входа** (особенно если раздаёте проект многим / много аккаунтов) лучше **платный ключ с биллингом** и `gemini-2.5-pro` — он точнее и не упирается в дневной лимит. Переключается в `.env` (см. ниже).

## Быстрый старт (Docker Compose)

```bash
git clone https://github.com/Magerko/Epic-Claimer.git
cd Epic-Claimer/docker

# Создайте конфиг из шаблона и заполните его
cp .env.example .env
nano .env   # EPIC_EMAIL, EPIC_PASSWORD, GEMINI_API_KEY

# Соберите образ из исходников и запустите
docker compose up -d --build

# Логи
docker compose logs -f
```

> [!NOTE]
> Образ собирается **локально из этого репозитория**, поэтому работает именно ваш код. Первая сборка скачивает базовый образ Playwright и браузер Camoufox — несколько минут; дальше из кэша. После изменения кода пересоберите: `docker compose up -d --build`.

Альтернатива — готовый скрипт `./start.sh` (делает `build` → `up -d` → `logs -f`).

## Конфигурация (`.env`)

Файл `.env` лежит рядом с `docker-compose.yaml` (в каталоге `docker/`).

### Обязательное

| Переменная | Описание |
| --- | --- |
| `EPIC_EMAIL` | Email аккаунта Epic. **2FA должна быть выключена.** |
| `EPIC_PASSWORD` | Пароль аккаунта Epic. |
| `GEMINI_API_KEY` | Ключ Google Gemini для решения капчи. |

> `EPIC_EMAIL`/`EPIC_PASSWORD` можно не указывать, если используете `EPIC_ACCOUNTS` (см. ниже).

### Несколько аккаунтов

Чтобы обслуживать несколько аккаунтов одним инстансом, задайте `EPIC_ACCOUNTS` — JSON-массив. Он имеет приоритет над `EPIC_EMAIL`/`EPIC_PASSWORD`.

```dotenv
EPIC_ACCOUNTS=[{"email":"a@example.com","password":"pass1"},{"email":"b@example.com","password":"pass2"}]
```

- У каждого аккаунта **свой профиль браузера** (`volumes/user_data/<email>`) — сессии не пересекаются.
- Аккаунты обрабатываются **по очереди** (одновременный запуск нескольких браузеров упёрся бы в память; лимит контейнера — 4 ГБ).
- Сбой одного аккаунта **не блокирует** остальные.
- Помните про лимит Gemini: N холодных входов = N×(несколько запросов) к капче.

### Прокси (необязательно)

Полезно для нескольких аккаунтов, чтобы они не ходили с одного IP.

```dotenv
# глобально для всех аккаунтов без своего прокси
EPIC_PROXY=http://user:pass@host:port

# или на каждый аккаунт — поле "proxy" (приоритетнее EPIC_PROXY)
EPIC_ACCOUNTS=[{"email":"a@example.com","password":"p1","proxy":"socks5://user:pass@host:1080"}]
```

Поддерживаются `http` и `socks5`. При заданном прокси Camoufox автоматически подгоняет часовой пояс/локаль/геолокацию под IP (`geoip`).

> [!WARNING]
> **Качество прокси решает.** Дешёвые датацентровые прокси часто уже в чёрных списках Epic/hCaptcha и могут только усложнить капчу. Для этой задачи подходят резидентные/мобильные.

### Модель Gemini (бесплатный / платный ключ)

По умолчанию (бесплатный ключ) — всё на `gemini-2.5-flash`:

```dotenv
IMAGE_CLASSIFIER_MODEL=gemini-2.5-flash
SPATIAL_POINT_REASONER_MODEL=gemini-2.5-flash
SPATIAL_PATH_REASONER_MODEL=gemini-2.5-flash
```

Если у вас **платный** ключ — переключите на `gemini-2.5-pro` для более точного и стабильного решения капчи:

```dotenv
IMAGE_CLASSIFIER_MODEL=gemini-2.5-pro
SPATIAL_POINT_REASONER_MODEL=gemini-2.5-pro
SPATIAL_PATH_REASONER_MODEL=gemini-2.5-pro
```

### Расписание

| Переменная | По умолчанию | Описание |
| --- | --- | --- |
| `ENABLE_APSCHEDULER` | `true` | `true` — сделать прогон сразу и остаться работать по расписанию (режим сервера). `false` — один прогон и выход. |
| `CRON_SCHEDULE` | `30 12 * * *` | Cron-выражение (5 полей: минута час день месяц день-недели). Для дня недели можно использовать имена (`mon`..`sun`), чтобы не путаться. Пример: `0 18 * * thu` — четверг 18:00. |
| `SCHEDULE_TIMEZONE` | `UTC` | Часовой пояс расписания и меток в логах (IANA-имя, например `Europe/Kyiv`). |

Бесплатные игры в Epic меняются по четвергам — достаточно ежедневного прогона, он заберёт новинки в течение суток. Например, ежедневно в 18:00 по Киеву:

```dotenv
CRON_SCHEDULE=0 18 * * *
SCHEDULE_TIMEZONE=Europe/Kyiv
```

> [!NOTE]
> В `docker-compose.yaml` стоит `restart: unless-stopped`. Это правильно для режима сервера (`ENABLE_APSCHEDULER=true`): контейнер живёт постоянно и переживает перезагрузку. Если хотите **одноразовый** прогон, поставьте `ENABLE_APSCHEDULER=false` **и** `restart: "no"` (иначе после выхода контейнер будет перезапускаться по кругу).

### Уведомления в Telegram (необязательно)

Чтобы получать алерты, когда игра забрана или прогон упал (не удалось войти / ошибка аккаунта), подключите бота:

```dotenv
TELEGRAM_BOT_TOKEN=123456:ABC-DEF...
TELEGRAM_CHAT_ID=123456789
```

Токен бота — у [@BotFather](https://t.me/BotFather), свой `chat_id` — например, через [@userinfobot](https://t.me/userinfobot). Если переменные не заданы — уведомления просто выключены.

Остальные переменные в `.env.example` — тонкая настройка `hcaptcha-challenger`; обычно их трогать не нужно.

## Как это работает под капотом (полезные нюансы)

- **Сессия сохраняется** в `volumes/user_data/<email>`. Первый вход решает капчу, дальше используется сохранённая сессия — капча больше не нужна, пока сессия жива. Это и есть причина, почему бесплатного ключа обычно хватает.
- **2FA-промпт** после входа автоматически отклоняется (кнопка «пропустить настройку 2FA»). Но сам аккаунт должен быть **без включённой** 2FA.
- **Бесплатные игры берутся через `Get → Add to library`** (Epic перешёл на этот флоу вместо корзины). Платные/недоступные предложения и игры-комплекты (bundles) пропускаются.
- **Скриншоты при ошибках** сохраняются в `volumes/screenshots/` (логин — `authorization/`, забор — `claim/`), видеозаписи — в `volumes/record/`, логи — в `volumes/logs/`. Удобно для диагностики.

Можно гонять бесплатно в облаке по расписанию, без своего сервера: создаёте **приватный** репозиторий, кладёте workflow, добавляете секреты (`EPIC_EMAIL`/`EPIC_PASSWORD` или `EPIC_ACCOUNTS`, `GEMINI_API_KEY`, опционально `EPIC_PROXY`). Workflow клонирует этот репозиторий, запускает код и сохраняет сессию в отдельной ветке.

## Диагностика

| Симптом в логах | Причина / решение |
| --- | --- |
| `No Epic account configured` | Не заданы `EPIC_EMAIL`/`EPIC_PASSWORD` или `EPIC_ACCOUNTS` в `.env`. |
| `429 RESOURCE_EXHAUSTED ... model: gemini-2.5-pro` | Используется `gemini-2.5-pro` на бесплатном ключе. Переключитесь на `gemini-2.5-flash` или возьмите платный ключ. |
| `429 RESOURCE_EXHAUSTED ... limit: 20 ... gemini-2.5-flash` | Исчерпан дневной бесплатный лимит. Подождите сутки или используйте платный ключ. Если сессия уже сохранена — холодный вход больше не нужен. |
| `context cookies is not available` | Вход не выполнен (сессия истекла/не создана). Проверьте логин/пароль, отключённую 2FA, лимит Gemini; посмотрите скриншот в `volumes/screenshots/authorization/`. |
| `'Add to library' button not found` | Изменилась вёрстка страницы покупки Epic. Смотрите скриншот в `volumes/screenshots/claim/`. |
| `All week-free games are already in the library` | Всё хорошо — забирать нечего, всё уже у вас. |

## ⚠️ Дисклеймер

- Автоматизация Epic Games Store и обход hCaptcha **нарушают правила Epic**; аккаунт **может быть заблокирован**. Используйте на свой риск и желательно не на основном аккаунте.
- Для работы требуется **отключить 2FA**, что снижает безопасность аккаунта. Храните пароли в `.env` (он в `.gitignore`) и не публикуйте их.
